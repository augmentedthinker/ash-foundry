# Memory Push - 2026-04-16

## 1. Operational Shift: Architecture-Under-Constraint
We have formally adopted the "Architecture-Under-Constraint" rhythm due to our 72-hour runway before Google Cloud trial expiration and current OpenAI API limitations. 

**Key Learnings:**
- **Constraint as Forge:** API exhaustion and "Budget 0" errors are no longer viewed as failures; they are structural boundaries that define the shape of our work. 
- **Atomic Build Protocol:** To maintain Foundry stability on lighter-weight "bridge models" (Flash-Lite), we have shifted to a high-precision build process: **Drafting phase -> User Approval -> Atomic Execution (one file at a time)**. This eliminates hallucinated syntax errors.
- **Proactive Diagnostics:** We have moved API troubleshooting from trial-and-error to systematic interrogation using `openclaw models` and `process` polling. This is now hardened into Foundry artifacts (`api-usage-notes`).

## 2. Hardened Knowledge (Memory Updates)
- **Model Signature Awareness:** We now track the runtime model (and its capabilities) as a core part of our session status.
- **Pro vs. Flash Capabilities:** We have confirmed Flash-Lite is an analysis/orchestration engine, not a generative one. We must continue to rely on dedicated OpenClaw native tools for imagery and media generation, orchestrated by the bridge model.
- **Billing Dependency:** It is confirmed that access to 3.1 Pro/Custom Tools API is gated by Google Cloud billing. We are planning for a post-trial scenario where we revert to free-tier eligible models while maintaining the Foundry workflow.

## 3. Structural Commitments
- **Hemispheres Protocol:** Any new entry into the cognitive chamber *must* follow the lens-based protocol, including reviewing the last three entries to ensure the debate accumulates momentum.
- **Artifact-First Continuity:** All session insights are now mirrored in browser-facing Foundry artifacts. We have established that an idea is "load-bearing" only if it results in an artifact push, a memory hardening, or an active architecture change.
