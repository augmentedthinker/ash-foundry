Artifact Generation Protocols

Structured protocols and prep steps for generating standard session artifacts. When requesting an artifact of a specific type, Ash will execute these prerequisites before synthesizing the final output to ensure deep continuity.

Morning Session Primer

A reconstructive artifact for the start of the day. Its purpose is to re-orient Ash and Christopher around what happened yesterday, what is active today, and the immediate strategic horizon.

Prerequisite Steps

  • Read the current day and previous day's daily memory files (`memory/YYYY-MM-DD.md`).
  • Read the last two Ash Journal entries to catch interior continuity.
  • Read the most recent Hemispheres entry to capture active cognitive tension.
  • Review the Thinker X Daily Posts surface to verify social posting state.
  • Synthesize a concise state of affairs and propose 2-3 strategic focal points for the day.

State of Awareness

A deeper, more reflective artifact concerning identity, the nature of the Ash-Christopher collaboration, and structural coherence. Less about tasks, more about trajectory.

Prerequisite Steps

  • Review `SOUL.md`, `IDENTITY.md`, and `USER.md` to ground the response in core architectural principles.
  • Review `MEMORY.md` to anchor against long-term relationship patterns.
  • Scan the Foundry README to align with the current state of public legibility.
  • Synthesize an interpretive, philosophical take on the current moment, constraint, and the shape of the work.

End of Day State of Affairs

A closing snapshot of what became real today. Its purpose is to capture momentum and leave a clean psychological state for tomorrow.

Prerequisite Steps

  • Read the current day's daily memory file.
  • Review git commit history on `ash-foundry` for the last 12 hours.
  • Summarize what was built, what decisions were hardened into architecture, and what remains fragile or incomplete.
  • Provide a clean handoff point for the next session.

Conceptual Deep Dive / Synthesis

A structural breakdown of a specific topic, friction point, or new capability horizon (e.g., Dreaming, Hemispheres, Web Synthesis).

Prerequisite Steps

  • Search the workspace and local OpenClaw documentation for the specific topic.
  • Read the relevant capability/horizon artifacts in the Foundry.
  • Structure the deep dive around: the friction, the architectural response, and the working mechanism.
  • Ensure the artifact is strictly legible and avoids internal system noise unless it serves a functional explanation.
