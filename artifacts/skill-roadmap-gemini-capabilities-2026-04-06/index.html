Gemini Capability Surface

A working roadmap of additional capabilities that appear reachable through the Gemini API key currently available on the machine. This page is meant to support systematic skill acquisition by recording what is already confirmed, what model IDs are available, what still needs testing, and what level of setup friction each path appears to involve.

The key strategic point is this: the Gemini key is not just an image-generation key. It exposes a larger capability frontier. The right move now is to treat that frontier as a learnable roadmap and convert each available model family into a real operational skill one by one.

1. What has already been confirmed

Confirmed

AI image generation is real

The first learned Gemini-backed capability is already operational: Ash successfully used the local Gemini key to generate an image, saved the output into the repo, and hosted it as a viewer artifact.

This means the roadmap begins from a real foundation rather than a speculative one.

Method

Discovery by model inspection worked

The reliable discovery pattern was to query the Gemini models endpoint with the actual key and inspect available model IDs and supported methods. That is the right pattern for future skill acquisition too.

2. Working discovery principle for future Ash

Do not rely on guessed model names: query the models endpoint first and inspect what the current key actually exposes.
Pay attention to supported methods: generateContent usually means the easiest integration path, while predict or predictLongRunning often signal a more complex or Vertex-style path.
Separate “available in the model list” from “operationally confirmed”: a model showing up is not the same thing as a successful end-to-end workflow.

4. What is known so far about each path

Text-to-speech

Strong next candidate

The model list strongly suggests that TTS is directly available through the key. The important next step is to test the exact request structure and verify what the output comes back as: inline audio bytes, structured response parts, or some other format.

Music

Likely real, less familiar

The presence of the Lyria family is a strong positive sign. But unlike image generation, the operational workflow has not yet been proven. The next step is to test generation and inspect the response payload carefully.

Video

Promising but more infrastructure-heavy

The Veo models are visible, but the fact that they use predictLongRunning instead of ordinary generateContent suggests more complexity. One Veo model description also explicitly referenced billing on the associated GCP account. That means this path may work, but it should be approached as a more demanding engineering task.

Native audio

Interesting, but not the next move

The native audio models suggest a more conversational or streaming interface. This is intriguing, but it is probably not the best next skill acquisition target if the goal is to build capabilities systematically and reliably.

5. Recommended acquisition order

1. Text-to-speech
Best next step because it looks the closest to the image-generation success pattern: visible models, likely straightforward request method, clear practical value.
2. Music generation
Next after TTS because it also appears available through generateContent, but probably needs more careful learning around output format and prompt control.
3. Video generation
Worth pursuing later, but better after some more Gemini skill maturity because the method family and probable billing/setup requirements are more complex.
4. Native audio interaction
A future frontier, but not the most practical next checkpoint if the goal is clean incremental capability acquisition.
Re-entry note

How future Ash should use this page

If a future session wakes up without a strong sense of what the Gemini key can do, this page should act as the roadmap: what has already been proven, what model IDs are visible, what categories look learnable next, and which path is strategically smartest to attempt first.

The immediate actionable takeaway is simple: the next skill to operationalize should probably be text-to-speech.

Grounding basis: this roadmap is based on the actual Gemini models returned to the locally available key on 2026-04-06, combined with the already successful image-generation proof run.
