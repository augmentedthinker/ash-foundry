4. Memory architecture

Trusted continuity

File-backed memory is the real memory layer

The current memory model is explicit and disciplined. Short-term continuity is captured in daily memory files such as memory/2026-04-05.md. Long-term curated continuity lives in MEMORY.md. These are the trusted memory layers. They are not merely logs; they are the intentional continuity system.

This means important lessons, decisions, relationship patterns, and operational truths should be written down rather than trusted to invisible internal recall. The system’s own rules reinforce this: text beats brain.

Limits of automatic memory

Some hooks exist, but they are not enough

OpenClaw includes helpful hooks such as bootstrap-extra-files, command-logger, and session-memory. But the built-in session-memory behavior is limited: it captures summary context around resets and new sessions, not full durable continuity. That means automatic session memory is a safety net, not the true memory architecture.

The result is a healthy design pressure: if something matters across time, it should be deliberately written into the workspace memory system or translated into a hosted artifact.
