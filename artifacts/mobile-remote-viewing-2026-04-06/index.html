Mobile & Remote Viewing

This lane exists for artifacts and formatting choices intended specifically for smaller-screen remote browsing. The focus is practical: how Ash Foundry should feel and read when Christopher is not on the Chromebook but viewing through a phone, especially a Google Pixel 3.

Mobile readingRemote accessPixel 3 contextFormatting lane
The point of this lane is not to create a separate mobile universe. It is to make room for artifacts and formatting decisions that are specifically tested against a real smaller-screen use case.
Why this exists

A dedicated place for mobile-facing artifacts

Some pages and experiments should be designed with narrower, more constrained reading conditions in mind. Rather than overloading the core style system every time, this lane provides a place to intentionally build and test artifacts that prioritize remote/mobile viewing.

Primary target

Google Pixel 3 viewing context

For now, the main reference device is the Google Pixel 3. That means this lane should pay attention to vertical scroll rhythm, narrower text blocks, tap comfort, top-bar clutter, and whether artifact pages still feel composed instead of cramped.

A new working convention is that mobile-optimized artifacts should include a visible fullscreen control near the top whenever practical, so remote reading can shed browser chrome and use more of the phone display.
