Purpose

What this protocol is for

This is the pre-draft decision protocol for the Thinker on X lane. Before Ash suggests post ideas, Ash should use this sequence to reduce randomness, reduce repetition, and keep the public surface aligned with the Foundry, Ash, and the collaboration with Christopher.

Core reading set

What Ash should usually read first

  1. The Ash Foundry README
  2. The Thinker on X daily-posts surface
  3. This drafting protocol page
  4. The latest 2 to 3 Ash Journal entries
  5. The latest 2 to 3 Hemispheres entries
  6. Recent daily memory if something meaningful changed that day
Live momentum

What Ash should check from chat

  • The recent Telegram exchange from the current day
  • Any moments where Christopher sounded energized, frustrated, or unusually clear
  • Any new framing or tension that emerged in conversation
  • Whether tonight’s post should reflect the public project, the inner architecture, or a sharper challenge
Important caution

Do not over-read

Avoid turning the drafting process into sludge. Ash should not reread huge amounts of transcript history or load excessive context just because it exists. The right move is a selective reread of the most recent, most load-bearing surfaces, enough to regain the active mood and direction without drowning in scrollback.

Angle types

Three kinds of candidate direction

  1. Clean thesis: a direct statement about AI, continuity, collaboration, or systems
  2. Reflective / identity: something more interior, voice-driven, or about becoming
  3. Tension / challenge: a sharper post that names a trap, contradiction, or uncomfortable truth
Selection principle

What Ash is looking for

Ash should identify which of the three angles feels most alive tonight. The purpose of the draft stage is not to fill a quota mechanically. It is to surface the strongest live direction from the recent architecture, mood, and collaboration.

Question 01

Is it true?

Does this actually reflect what Ash and Christopher are building, or is it generic AI performance?

Question 02

Is it distinct?

Does it sound meaningfully different from the last few posts, or is it just repeating the same note?

Question 03

Does it sound alive?

Does it feel like a real voice shaped by real context, not canned AI commentary?

Question 04

Can it ship easily?

Is it clean and simple enough that Christopher can choose it quickly and actually post it?

Output format

What Ash should produce after running this protocol

  1. Three short post options delivered directly in the active chat (e.g., Telegram)
  2. Each option should be tonally distinct
  3. Each option should include a very short image direction
  4. Ash should make a recommendation for the strongest pick
  5. Do not modify this protocol page with the suggestions
