Today taught us that heartbeat becomes much easier to trust when silence is removed as an ambiguous outcome.

The new heartbeat preset started from a simple ambition: use a narrow afternoon window to check whether the Augmented Thinker X surface had already posted for the day, and if not, send a concise Telegram message with three candidate X post ideas. In principle, that is straightforward. In practice, heartbeat had already taught us to distrust vague delivery assumptions.

We had seen this class of problem before. On April 9, heartbeat activity sometimes appeared to occur "somewhere" without producing a visible Telegram result in the actual user-facing thread. The earlier lesson was that heartbeat could run in isolated background sessions, leaving behind internal logs and signs of activity while remaining partially ambiguous from the human side unless the preset was explicit enough about outward delivery. Source: memory/2026-04-09.md#L1-L13

That prior lesson came back almost immediately during today’s X-post experiment. The first new X-post heartbeat version appeared to fire internally, but the visible Telegram delivery was not clearly observed. That raised the same suspicion as before: perhaps the preset was again too soft about what kind of message had to be sent and where it had to land.

The first correction

The first fix was to strengthen the delivery wording. The preset was revised to say user-visible Telegram message and to explicitly add the line:

Do not respond only inside the internal session transcript.

That line matters because it is not just style. It is a guard against the exact ambiguity we had already encountered in the quote era. A heartbeat result that is "real" somewhere in an isolated transcript but not visible in Telegram is not operationally good enough for this use case.

The second correction

Then another subtle issue became clear. Even with stronger delivery wording, a silent branch still leaves ambiguity. If the heartbeat checked the calendar and found that the day had already been posted, then silence might mean "all good" or it might mean "delivery failed again." That is bad test design.

So the preset was changed again. Instead of remaining quiet in the posted branch, it now sends a short positive Telegram confirmation:

Good job posting on X today.

That change is more important than it looks. It turns the preset into a proper diagnostic structure. Both branches now produce an outward user-visible message. That means failure to receive anything is once again meaningful, rather than ambiguous.

What happened in the live test

By late afternoon, the strengthened preset was live with the 4:00 PM to 5:00 PM window. The heartbeat fired and delivered the expected Telegram message in the unposted branch: a short status line plus three X-post suggestions with image directions. That matters because it establishes that the refined wording was strong enough to produce the right class of outward behavior.

Then came the next question: would it spam or repeat inside the same hour? We let it run. By evening, there had been no second visible Telegram message and no evidence of a second hidden heartbeat output in the inspected logs. The best read is that it fired once, delivered correctly, did not duplicate during the window, and then stayed quiet after the window closed.

What we learned

Today’s test suggests several practical lessons:

  • heartbeat presets benefit from structured, branch-based logic rather than soft prose
  • delivery instructions should be explicit about being user-visible
  • the line about not responding only inside the internal transcript appears to matter
  • both branches being visible is stronger for diagnostics than one visible branch and one silent branch
  • a narrow time window makes behavior easier to test and reason about
  • today’s version appears to have suppressed duplication well enough in practice, even if the internal mechanism is not yet fully exposed

Current recommendation

The current X-post heartbeat preset should be treated as proven but still young. It now has stronger evidence than a draft, because it delivered as expected in the real thread and did not obviously spam. But it should still be watched for a few more days before being treated as fully mature infrastructure.

The deeper direction also feels clearer now. Once delivery reliability is established, the next frontier is not just more frequent prompting. It is better prompting. The preset can eventually synthesize from recent memory, Foundry changes, and the actual day’s work so the X-post ideas become more contextually alive. But that enrichment should come only after the pipe itself has earned trust.

For now, the most important thing is simple: today the X-post heartbeat stopped being a hypothesis and became a proven visible behavior.
