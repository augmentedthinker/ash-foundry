Foundry Glass

A deliberately different visual direction for Ash Foundry: bright, airy, translucent, more modern-app than forge-chamber. Same structural patterns, very different atmosphere.

Alternate styleLight glassmorphismSame layout grammarVisual contrast test
Foundry Glass keeps the site’s structural language but swaps the mood completely: less ember-and-metal, more daylight-and-frosted panels. It is useful precisely because it proves the system can now support aesthetic replacement without redesigning every page from scratch.

How it differs from Foundry Ember

Mood shift

Bright instead of molten

Foundry Ember feels dark, grounded, and heat-coded. Foundry Glass is intentionally much lighter: cool whites, soft blues, translucent surfaces, and a cleaner product-design atmosphere.

The point is to test contrast, not just variation. This should feel like a distinct design family, not a mild tint adjustment.

Structural continuity

Same site grammar, new aesthetic skin

The underlying page architecture remains familiar: hero blocks, cards, quotes, pill rows, details dropdowns, and content grids. That preserves readability and makes style comparison easier.

In other words, the bones stay stable while the visual skin changes meaningfully.

Why this matters

It proves swapability: the repo can now host multiple visual systems without reauthoring every page from zero.
It creates a real comparison set: now there is an initial style and an alternate style that are actually different enough to evaluate.
It sharpens design language: once styles are named and embodied, it becomes much easier to talk about what the site should feel like.
It fits the new hierarchy: site.css can later be pointed at Glass if this style becomes the active default for the whole site.
Named implementation: assets/css/foundry-glass.css
Relationship to default: available as an alternate option beneath site.css
